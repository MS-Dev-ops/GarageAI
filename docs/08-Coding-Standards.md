\# Coding Standards



Version: 1.0



\---



\# Principles



Readable code is more valuable than clever code.



\---



Prefer



Small classes



Small methods



Single responsibility



Constructor Injection



Immutable request models



\---



Avoid



Static helpers



God classes



Magic strings



Duplicate logic



Premature abstraction



\---



Every class should answer



"What is my single responsibility?"



If the answer requires "and",



the class probably has multiple responsibilities.



\---



Prefer composition over inheritance.



\---



Use official SDKs.



Avoid unofficial wrappers unless justified.



\---



Every architectural decision must have a reason.



Every abstraction must earn its existence.
